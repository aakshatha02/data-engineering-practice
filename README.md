*This data engineering practice from (https://github.com/danielbeach/data-engineering-practice), it helped me understand handling files. The high level topics that these practice problems is covered with.*

- Python data processing.
- csv, flat-file, parquet, json, etc.
- SQL database table design.
- Python + Postgres, data ingestion and retrieval.
- PySpark
- Data cleansing / dirty data.

**Exercise 1 - Downloading files**<br>
To download a number of files from an HTTP source and unzip them, storing them locally with Python.

**Exercise 2 - Web Scraping + Downloading + Pandas**<br>
To perform web scraping, build uris, download files, and use Pandas to do some simple cumulative actions.

**Exercise 3 - Boto3 AWS + s3 + Python**<br>
Used popular aws package called boto3 to try to perform a multi-step actions to download some open source s3 data files.

**Exercise 4 - Convert JSON to CSV + Ragged Directories**<br>
Focused more file types json and csv, and working with them in Python. Traversed a ragged directory structure, finding any json files and converting them to csv.

**Exercise 5 - Data Modeling for Postgres + Python**<br>
Created a data model / schema to hold these data sets, including indexes, then create all the tables inside Postgres by connecting to the database with Python.

**Exercise 6 - Ingestion and Aggregation with PySpark**<br>
Loaded some files using PySpark and then be asked to do some basic aggregation.

**Exercise 7 - Using Various PySpark Functions**<br>
Focused on using a few of the more common build in PySpark functions pyspark.sql.functions and applying their usage to real-life problems.

**Exercise 8 - Using DuckDB for Analytics and Transforms**<br>
This exercise is on analytical and transformation tasks using DuckDB. This will require an understanding of the functions and documenation of DuckDB.

**Exercise 9 - Using Polars lazy computation**<br>
The ninth exercise based on Polars, is a new Rust based tool with a wonderful Python package that has taken Data Engineering by storm. It's better than Pandas because it has both SQL Context and supports Lazy evalutation for larger than memory data sets!

**Exercise 10 - Data Quality with Great Expectations**<br>
The tenth exercise This exercise is to helped me learn Data Quality, specifically a tool called Great Expectations. Implemented Data Quality checks to catch some of these issues.
